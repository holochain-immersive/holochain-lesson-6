Validation Rules

### Validation Workflow - After an action is committed, it is transformed into different DHT Operations (DHT Ops) - Then, those DHT Ops are published to the different neighborhoods in the DHT - When those nodes receive that publish, they add it to their validation queue - If the validation fails, those nodes produce warrants that get gossiped throughout the network - All other nodes can verify that the bad agent is behaving badly - They then disconnect from the bad agent - If the validation succeeds, the node starts to serve that record to DHT queries (get, get_links, etc.)

### System Validation - Holochain enforces some system level validations across all apps - Source chains can't be forked - All signatures must be signed by their author - Timestamp and sequence number for the actions must be incremental - Zome ID
### Integrity Zomes - Integrity zomes declare which moves are valid and not valid in a happ - Callback: `validate(dht_op: Op)` - Three results possible: 1. Valid 2. Invalid: it specifies the reason why it failed 3. Unresolved dependencies: Not enough data to complete the validation - Must be deterministic - There are special gets: `must_get` - `get_links` is forbidden - `get_agent_activity` bounded by action hashes
### Validation callback function context - The function is called in two contexts 1. When committing an action, the author itself will run the validation callback to make sure their commits follow the rules 2. When receiving a publish for a DHT Op, the receiving agent will run the validation callback. If it returns valid, then that node starts serving that DHT Op
### Validation challenges - Things that can't be considered during validation, without "____ing" - links - updates - mutes - Challenging because these items are inconsistent over time, and therefore can't be collected deterministically/consistently by two different nodes - "___ing" - Deterministically defining some set of actions (through a combination of naming their hashes explicitly, and following the named hashes they reference) to discover the links, updates, and mutes
